The logical type

Boolean is a data type that has only two possible values: true and false. It is also known as the logical type.

val t = true  // t is true
val f = false // f is false

println(t) // true
println(f) // false

Note that you cannot assign an integer value to a Boolean variable. In Kotlin, 0 is not the same as false.

This type is often used in programming (not only in Kotlin) to represent two opposite states such as:

  • a store is open (open is true, closed is false)
  • a plane is flying (yes is true, no is false)
  • travel is allowed (yes is true, no is false)
  • an option is enabled (yes is true, no is false)
  • and so on.

Reading Boolean values

With Kotlin 1.4 you can read a Boolean value like this:

val b: Boolean = readLine().toBoolean()

The toBoolean() returns true if the input is "true", case insensitive, otherwise false.

A small example, suppose you have this input:

true
True
TRuE
T
false

Let's see how this input is read and converted:

println(readLine().toBoolean()) // true
println(readLine().toBoolean()) // true
println(readLine().toBoolean()) // true
println(readLine().toBoolean()) // false
println(readLine().toBoolean()) // false

With Kotlin below 1.3 you need to use readLine()!!:

val b: Boolean = readLine()!!.toBoolean()

You can use readLine()!!.toBoolean() with any compiler version. Although in Kotlin 1.4 it's not necessary.

Logical operators

Variables of the Boolean type are often used to build logical expressions with logical operators. Kotlin has four logical operators NOT, AND, OR and XOR:

  • NOT is a unary operator that reverses the Boolean value. It is denoted as !.
val f = false // f is false
val t = !f    // t is true
  • AND is a binary operator that returns true if both operands are true, otherwise, it returns false. It is denoted as &&.
val b1 = false && false // false
val b2 = false && true // false
val b3 = true && false // false
val b4 = true && true  // true
  • OR is a binary operator that returns true if at least one operand is true, otherwise, it returns false. It is denoted as ||.
val b1 = false || false // false
val b2 = false || true  // true
val b3 = true || false  // true
val b4 = true || true   // true
  • XOR (exclusive OR) is a binary operator that returns true if the Boolean operands have different values; otherwise, it is false
val b1 = false xor false // false
val b2 = false xor true  // true
val b3 = true xor false  // true
val b4 = true xor true   // false

People don't use the XOR operator as often as the other logical operators. Just remember that Kotlin has it. If you really need it, you can use it.

Precedence of logical operators

Below you can see the hierarchy of logical operations:

  • ! is NOT
  • xor is XOR
  • && is AND
  • || is OR

So, the variable below is true:

val bool = true && !false // true, because !false is evaluated first

To change the order of execution you can use parentheses (...).

Example: Hiking

As an example, let's write a Boolean expression that determines the possibility of trekking during summer and during other seasons.

val cold = false
val dry = true
val summer = false // suppose now is autumn

val hiking = dry && (!cold || summer) // true, let's go hiking!

Do not get confused by the expression above; otherwise, you will go on a hike when it's raining :) A programmer should understand not only arithmetic, but also logical operations.
