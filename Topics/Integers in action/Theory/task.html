Basic information about Int and Long types

Integer numbers are often used in real programs. As you may recall, Kotlin has several types to store them. The most used types are Int (32 bits) and Long (64 bits). You can perform all types of arithmetic operations (+, -, *, /, %) with variables of these types. Let's take a look at several examples.

val two = 2  // Int
val ten = 10 // Int

val twelve = two + ten // 12
val eight = ten - two  // 8
val twenty = two * ten // 20
val five = ten / two   // 5
val zero = ten % two   // 0, no remainder

You may already know what these operations actually do. The same operations work with the Long type. To declare a variable of the Long type, you may choose one of these ways:

val longNumber1 = 1_000_000_000_000_000
val longNumber2: Long = 1_000_000
val longNumber3 = 1000L

val result = longNumber1 + longNumber2 - longNumber3
println(result) // 1000000000999000

The value longNumber1 is Long because it is greater than the possible value of the Int type. The value longNumber2 is Long because we specified the type. The value longNumber3 is Long because we tagged the value with the suffix L (Long). The result is Long because it is the result of the sum of three Longs.

Do not use Longs unless you absolutely have to (e.g., to process large values).

See those underscores ( _) in numbers? In Kotlin, you can use underscores to simplify the readability of a number. We often group digits using them. Underscores can't be placed before or after a number, but they can be placed anywhere inside it and even in a row.

Reading numbers from the standard input

As a rule, to solve some problem you need to read some data from the world outside, process it, and output the result. For example, this program reads two numbers from the standard input, adds them, and outputs the sum.

fun main() {
    val a = readLine()!!.toInt()
    val b = readLine()!!.toInt()

    val sum = a + b

    println(sum)
}

This simple code uses readLine() to read data. It works for positive, negative, and zero numbers because the Int type supports all of them.

If we know that the input numbers can be quite large, we can read Long's instead of Int's:

val a = readLine()!!.toLong()
val b = readLine()!!.toLong()

No more lines need to be changed in this code.
